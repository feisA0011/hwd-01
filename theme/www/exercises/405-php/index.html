Exercise 405 - Janitor and Detector

This exercise is about developing your first Janitor based PHP project

The purpose of this exercise is to work with your personal Janitor project to gain experience with Janitor and Detector segments.

Tasks

  1. Understand the Detector segments

    Janitor natively supports Detector segments. While Janitor can also be used entirely without segments it is recommended to use the segments to provide optimized UI's for each segment, rather than one size fits all.

    Janitor will automatically detect the segment of the visiting browser, by making a request to the Detector API, when the $page->segment method is called. As default this happens in the templates/www.header.php template, where the segment of the current user's browser is used to include the correct JavaScript and CSS. The detection process only happens once per site visit and the detected segment is stored in the user session for the duration of the visit.

    In addition Janitor accepts a dev=1 GET parameter on any page, which will force the site to load the development code, rather than the generated production code. This parameter simply indicates whether to include the system generated JavaScript and CSS bundles or the individual source files. Use dev=1 when developing for better debugging options.

    You will learn more about how to structure your JavaScript and CSS in the following exercises.

  2. Defining the segment support

    The exact segment support can be individualized for each Janitor project. In theme/config/segments.php you will find the settings for your current project in an associative Array. You will find that the keys represent the available segments, and the values specify which segment will be returned, when a browser from any of the segments visits your page.
