Exercise 316 - JS, Manipulator framework 1

This exercise is about using the Manipulator framework.

The purpose of this exercise is to practice using a framework to structure your application of code.

Use the script tag in the bottom of the exercise page to write your JavaScript.

The manipulator library is included in this page. You can use everything you find in the Manipulator library documentation.

Tasks

  1. Investigate the code

    In index.html you will find a small, regular snippet of HTML. Note that the div.scene has an opacity:0 setting in the CSS in the top of the page, so it will be invisible from the beginning.

    In js/i-page.js you will find a simple page controller. It's init method will automatically be called when the DOMContentReady event occurs. It will output an "init page" text in the console. This happens because of the DOMReady event listener/handler in the bottom of the i-page.js file â€“ AND the page initializer classname (i:page) on the div#page in index.html. If you remove the initializer classname on the div#page, then the init method will not be called. Try to remove the initializer classname and observe the result. Remember to put it back before you continue.

    In the page initializer, three new methods are created for the page object (div#page), resized, scrolled and ready. The ready method will be called instantly and the resized method will be invoked on page resize and scrolled on page scroll, due to the two event listeners applied in the ready method.

    Finally the ready method defines a reference to the current scene, which can be used later in your code.

  2. Make the page resize handler

    The scene should always be as high as the browser. Set the scene height in page.resized.

  3. Show the page

    The scene should be shown only when the page is ready. Add a transition and change the opacity to 1 to make the page fade up, when it is ready.
